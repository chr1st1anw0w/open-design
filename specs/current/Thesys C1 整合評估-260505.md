## 工具核心能力概覽

以下是針對 **Thesys C1 SDK** 整合至你的 **Open Design（`gpt-image2` workbench / prompt-studio）** 的完整評估分析：

**Thesys C1** 是一個 Generative UI API middleware，核心作用是將 LLM 的文字輸出轉換為即時的互動式 React 元件（表單、表格、圖表、卡片、投影片等）。其架構採用 OpenAI 相容端點，任何現有使用 OpenAI API 的後端幾乎無需修改即可接入 。前端透過 `@thesysai/genui-sdk` 中的 `<C1Component>` 接收結構化 DSL 字串並渲染為 React 元件樹，整個流程支援 Streaming 。thesys+2

底層 UI 基礎建立在 **Crayon**（Thesys 的開源設計系統，基於 Radix UI primitives + shadcn/ui 模式）之上 ，並透過 `<ThemeProvider>` 將 design tokens 注入整個元件樹，使樣式可完全客製化 。abduzeedo+1

------

## Open Design 現況分析

從你的 **Workbench** 頁面內容來看，Open Design 目前的架構是：

* **Prompt Studio**：靜態模板選擇介面（UI/UX、academic figures、branding 等 70+ 分類），用戶選擇模板後生成提示詞[thesys](https://docs.thesys.dev/react-reference/c1-chat)
* **Workbench**：提示詞定製化工作臺，目前為表單/選單驅動的靜態邏輯
* **技術棧**：Next.js（頁面底部可見 Open Next.js Dev Tools）[thesys](https://docs.thesys.dev/react-reference/c1-chat)

目前 AI 助手對話元件的呈現方式推測為純文字或基礎 Markdown，缺乏動態 UI 回應能力。

## 技術相容性

| 評估面向 | Open Design 現況          | Thesys C1 需求                            | 相容度     |
| -------- | ------------------------- | ----------------------------------------- | ---------- |
| 框架     | Next.js                   | Next.js / Vite+React / Cloudflare Workers | ✅ 完全相容 |
| LLM 後端 | GPT-image2（推測 OpenAI） | OpenAI 相容端點（可接 Claude、Gemini）    | ✅ 直接接入 |
| 元件系統 | 自定義 React 元件         | 可注入 Custom Components                  | ✅ 可擴充   |

## 整合優先建議場景

**Prompt Studio 的 AI 助手對話升級**是最高優先項目。現在的提示詞建議可能是靜態文字輸出；接入 `<C1Chat apiUrl="/api/chat" />` 後，AI 回應可直接渲染為互動式表單（讓用戶在對話中填寫參數）、動態預覽卡片或多步驟引導流程 。這對提示詞調整體驗的提升非常顯著。[thesys](https://docs.thesys.dev/guides/custom-components)

**Workbench 的模板定製邏輯**可透過 Custom Components 強化 。例如：將現有靜態 `<select>` 模板選擇器，替換為 C1 生成的動態篩選面板，讓 AI 根據用戶輸入的風格關鍵字即時推薦適合的模板類別，並以卡片式 UI 呈現，而非純文字清單。[thesys](https://docs.thesys.dev/guides/migrate-to-genui)

## 具體整合路徑

## Step 1：後端接入（最低成本）

在 Next.js API Routes（`/api/chat`）修改 OpenAI client 指向 Thesys 端點：

```
typescript
import OpenAI from 'openai';

const client = new OpenAI({
  apiKey: process.env.THESYS_API_KEY,
  baseURL: 'https://api.thesys.dev/v1/embed',
});
```

由於採用 OpenAI 相容格式，Open Design 現有的 GPT-image2 呼叫邏輯幾乎不需改動 。[thesys](https://www.thesys.dev/blogs/how-to-build-generative-ui-applications)

## Step 2：前端 SDK 安裝與 ThemeProvider 注入

```
bash
npm install @thesysai/genui-sdk @crayonai/react-ui
```

在 `layout.tsx` 最外層包裹 `<ThemeProvider>`，並注入符合 Open Design 現有設計語言的 design tokens 。這裡需要做的主要工作是 **mapping 現有 CSS 變數到 Crayon token 體系**，避免樣式衝突。[thesys](https://docs.thesys.dev/guides/rendering-ui)

## Step 3：對話元件替換策略

針對 **Prompt Studio** 的 AI 助手，建議使用 `<C1Chat>` 元件搭配 `customizeC1` prop 注入現有客製元件，而非完全重寫：

```
tsx
import { C1Chat } from "@thesysai/genui-sdk";
import { PromptPreviewCard, TemplateSelector } from "./components";

export default function PromptStudio() {
  return (
    <C1Chat
      apiUrl="/api/chat"
      formFactor="panel"   // 適合側欄式對話
      theme={{ ...themePresets.slate, mode: "dark" }}
      customizeC1={{
        customComponents: { PromptPreviewCard, TemplateSelector },
      }}
    />
  );
}
```

這樣可讓 AI 在對話中直接渲染 Open Design 自有的提示詞預覽卡片 。[thesys](https://docs.thesys.dev/guides/migrate-to-genui)

## 風險與注意事項

**樣式橋接複雜度**是主要風險點。Crayon 基於 Radix UI，若 Open Design 已有大量自定義 CSS，`ThemeProvider` 的 CSS 變數可能與現有 class 衝突 。建議先在獨立的 `/tools/gpt-image2/chat-test` 路由做隔離測試，確認主題注入正確後再擴展到 Workbench 和 Prompt Studio。[abduzeedo](https://abduzeedo.com/thesys-generative-ui-c1-api)

**API 費用結構**需要評估：Thesys C1 目前提供 $10 免費額度，之後按 token 計費，與 GPT-image2 的計費模型分開計算 。如果 Open Design 的對話頻率高，需要預估月度成本是否合理。[comparateur-ia](https://comparateur-ia.com/en/reviews/thesys)

**Artifacts API** 值得特別關注——C1 可以直接在對話中生成完整的投影片和報告，對 Prompt Studio 的「輸出展示」場景有很大潛力，可在後續規劃 Phase 2 整合 。[comparateur-ia](https://comparateur-ia.com/en/reviews/thesys)
