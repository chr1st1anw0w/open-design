概念說明

什麼是 DESIGN.md？

DESIGN.md 是 Google Stitch 提出的概念——一個純文字 Markdown 設計系統文件，專門讓 AI Agent 讀取後生成風格一致的 UI。它不需要 Figma 匯出、JSON Schema 或任何特殊工具，直接放進專案根目錄即可使用。

文件誰讀取定義內容舉例
AGENTS.mdCoding Agents如何建構專案（技術架構、工具鏈）使用 Next.js + Tailwind，禁止 jQuery
DESIGN.mdDesign Agents / AIUI 如何呈現（顏色、字型、元件樣式）主色 #01696f、按鈕圓角 8px、Satoshi 字型
README.md開發者專案說明、如何啟動npm install && npm run dev

每個 awesome-design-md 的模板包含三個檔案：DESIGN.md（設計系統主文件）、preview.html（Light Mode 視覺預覽）、preview-dark.html（Dark Mode 視覺預覽），讓你在使用前可以先確認風格是否符合需求。

詳細步驟

逐步安裝說明

以下五個步驟帶你從零開始，將 awesome-design-md 模板應用到新專案。

1
瀏覽 & 選擇模板

前往 GitHub 瀏覽 58 個設計模板

進入 design-md/ 資料夾，每個子目錄對應一個網站的設計系統。建議先在瀏覽器打開 preview.html 或 preview-dark.html 確認視覺風格，再決定要使用哪個模板。

推薦從這幾個開始： vercel/（極簡黑白）、linear.app/（紫色精準感）、notion/（暖色調極簡）、figma/（鮮豔多色）
2
取得模板 — 方法 A

Clone 整個 Repository（推薦首次使用者）

將整個倉庫 Clone 到本地，方便離線瀏覽所有 58 個模板並快速比較。

# Clone 整個倉庫到本地
git clone https://github.com/VoltAgent/awesome-design-md.git

# 進入目錄
cd awesome-design-md

# 查看所有可用模板
ls design-md/
Clone 後可在瀏覽器直接打開各資料夾的 preview.html 預覽視覺效果，再決定複製哪個 DESIGN.md。
3
取得模板 — 方法 B

直接下載單一 DESIGN.md（輕量快速）

若只需要特定一個模板，可以直接用 curl 或 wget 下載原始 Markdown 檔案。

# 以 Vercel 風格為例，直接下載到當前專案根目錄
curl -o DESIGN.md   https://raw.githubusercontent.com/VoltAgent/awesome-design-md/main/design-md/vercel/DESIGN.md

# 或使用 wget
wget -O DESIGN.md   https://raw.githubusercontent.com/VoltAgent/awesome-design-md/main/design-md/linear.app/DESIGN.md

# 其他熱門模板 URL 格式
# .../design-md/notion/DESIGN.md
# .../design-md/figma/DESIGN.md
# .../design-md/stripe/DESIGN.md
也可以在 GitHub 網頁上找到 DESIGN.md 檔案，點擊「Raw」按鈕，然後 Cmd+S 儲存到專案根目錄。
4
整合到新專案

將 DESIGN.md 放入專案根目錄並依需求修改

確保 DESIGN.md 位於你的專案根目錄（與 package.json 同層）。然後用任何文字編輯器打開，修改品牌色彩、字型名稱等關鍵 token。

# 你的專案結構應該如下
my-project/
├── DESIGN.md          ← 放在根目錄
├── AGENTS.md          ← (選填) AI 建構指引
├── package.json
├── src/
│   ├── components/
│   └── pages/
└── README.md

# 如果是從 Clone 的倉庫複製過來
cp ~/awesome-design-md/design-md/vercel/DESIGN.md ./DESIGN.md
常見需要修改的項目： 品牌主色 hex 值、字型名稱（如換成自己已授權的字型）、公司或專案名稱、特定元件的圓角大小。
5
告知 AI Agent

在 Prompt 中引用 DESIGN.md 讓 AI 生成對應風格

只要告訴你的 AI Coding Agent（Claude / Cursor / GitHub Copilot）參照 DESIGN.md，它就會自動讀取並生成風格一致的 UI 程式碼。

-- Claude Code / Cursor 的 Prompt 範例 --

"Build me a dashboard page following the design system 
defined in DESIGN.md. Include a sidebar navigation, 
KPI cards, and a data table."

-- 或更精確的指令 --

"Create a login form component using the color tokens, 
typography scale, and button styles from DESIGN.md. 
Use the primary button style for the submit action."

-- 搭配 Agent Prompt Guide（DESIGN.md 第 9 節）--

"Use the quick color reference in Section 9 of DESIGN.md 
to style this card component."
每個 DESIGN.md 的第 9 節（Agent Prompt Guide）都包含已整理好的顏色速查表和現成可用的 Prompt 模板，可以直接複製使用。
檔案結構

DESIGN.md 內容結構

每個 DESIGN.md 遵循 Google Stitch 格式並有擴充，包含 9 個標準化章節，讓 AI 能精確理解每個設計決策。

#章節名稱包含內容
1Visual Theme & Atmosphere整體氛圍、設計哲學、密度感（例如：極簡、技術感、溫暖）
2Color Palette & RolesSemantic 顏色名稱 + HEX 值 + 功能角色（primary, error, warning…）
3Typography Rules字型家族、完整字型階層表、字重規則
4Component Stylings按鈕、卡片、輸入框、導航列各狀態（hover, active, disabled）
5Layout Principles間距比例尺、格線系統、留白哲學
6Depth & Elevation陰影系統、表面層次（surface hierarchy）
7Do's and Don'ts設計護欄與反模式清單（避免使用的顏色組合、元素等）
8Responsive Behavior斷點設定、觸控目標大小、摺疊策略
9Agent Prompt Guide顏色速查表、即用 Prompt 模板（重要：AI 最常參考此節）

使用技巧

進階使用技巧

🔀

混搭不同模板

可以取 Vercel 的顏色系統、Linear 的元件規範、Notion 的字型規則，手動組合出符合品牌的 DESIGN.md。

📌

加入 AGENTS.md

同時在根目錄放 AGENTS.md 定義技術架構（框架、禁用函式庫等），讓 AI 同時理解設計與技術限制。

🎯

引用第 9 節 Prompt Guide

DESIGN.md 的最後一節是專為 AI 整理的速查表，Prompt 中明確指向 "Section 9" 可提升生成精準度。

👁️

先看 preview.html

Clone 後打開各模板的 preview.html，用瀏覽器直接確認視覺效果，比讀 Markdown 文字更直觀快速。

✏️

維護版本控制

修改後的 DESIGN.md 納入 Git 版本控制，方便團隊協作時每位成員都參照相同的設計基準。

🔁

反饋修正循環

AI 生成後若風格不符，在 DESIGN.md 中修正對應 token（如調整主色或陰影），再重新請 AI 生成直到滿意。
